FAQ

Frequently asked questions about the FedComp Index

What is FedComp Index?

An open-source competitive instrumentation system for federal contractors. It ingests five years of obligation data from USASpending.gov, joins it against SAM.gov registration records, and produces a FedComp Index score (0–100), posture class assignment, proximity map, and public contractor dossier for every scored firm in a state.

Nevada is the initial coverage area with {{ stats.total }} scored contractors.

What is a posture class?

A trichotomous classification derived from the FedComp Index score, partitioning contractors by demonstrated award capacity:

  • Posture Class 1 - Score 60+. Approximately $100M+ in cumulative obligations over the trailing 5-year window.
  • Posture Class 2 - Score 40–59. Approximately $5M–$100M. Sustained mid-magnitude performers.
  • Posture Class 3 - Below 40. Nascent or sub-threshold obligors.

Thresholds are fixed and state-invariant. Posture class is orthogonal to SBA certifications, set-aside eligibility, and firm size standards, all of which are population-dependent classifications that shift with regulatory revision rather than measuring demonstrated obligation capacity. Movement between classes occurs only when the underlying FedComp Index score crosses a boundary.

Posture class thresholds and scoring formula are specified on the methodology page.

How is the FedComp Index score calculated?

Composite of two index drivers. No normalization, no percentile ranking. The FedComp Index score is absolute: a contractor scoring 60 represents equivalent obligation magnitude regardless of jurisdiction.

Award Volume (0.90 weight): log10 of total federal obligations over the trailing 5-year window, monotonic transform to [0, 100]. Reference points: $10K → 0, $1M → 33, $10M → 50, $100M → 67, $1B → 83.

Award Recency (0.10 weight): temporal step function on last award date. 0–12 months = 100, 12–24 = 60, 24–36 = 30, 36–60 = 10. Position drift of up to 9 points as the recency bucket decays.

Composite: (V × 0.90) + (R × 0.10), clamped to [0, 100]. Full specification: fedcompindex.org/methodology.

What is the proximity map?

A directed graph resolving the six contractors nearest in competitive topology. Edges weighted by shared NAICS and PSC codes from awarded contracts using inverse document frequency (1/n2 where n = obligors holding that code). PSC codes carry 2× base weight. Scale-filtered by (min/max)2 of total award volumes; scale-disjoint pairs converge to zero.

The proximity map is asymmetric. Contractor A appearing in B's competitive radius does not guarantee the converse. Firms with broad code coverage appear in many smaller firms' neighborhoods; their own proximity map contains only comparably diversified obligors.

What is a contractor dossier?

The public instrument readout for a single firm, accessible at fedcompindex.org/dossier/[contractor-name]/ or via site search. Contents: FedComp Index score decomposed into constituent index drivers, full obligation history (dates, agencies, NAICS codes, obligated amounts), proximity map, SBA certifications, and spectator count.

Each contractor dossier generates a posture card: a single-image distillation of the firm's FedComp Index score, posture class, state rank, and cumulative obligation value. The posture card serves as the OG/meta image when the dossier URL is shared on LinkedIn, Slack, or X, and embeds in capability statements and teaming packages. Dossier URLs are permanent; data refreshes daily.

How does FedComp Index differ from SAM.gov and USASpending?

SAM.gov is a registration ledger: entities and their self-reported attributes. USASpending.gov is a transaction ledger: obligations, recipients, instruments. Neither scores, ranks, or resolves competitive adjacency.

FedComp Index joins both ledgers and applies a scoring function and topological analysis that neither source provides. The underlying obligation data is identical to what any analyst can query from the public APIs. The instrument layer (FedComp Index score, posture class, proximity map, contractor dossier) is the differentiator.

What is the Tabularium?

The instrument archive of FedComp Index: datasets on HuggingFace and Kaggle (CSV, Parquet), scoring engine on PyPI (fedcomp-index-scoring), pre-scored bundles (fedcomp-index-data), and source repository on GitHub. All artifacts MIT-licensed.

Named after the Roman Tabularium on the Capitoline Hill where tabulae publicae (public records and legal instruments) were deposited.

Where can I download the data?

  • HuggingFace - Nevada contractors
  • Kaggle - Nevada contractors
  • PyPI: pip install fedcomp-index-data

Key columns: fedcomp_index (0–100), posture_class (Class 1/2/3), awards_5yr_total_usd, award_count, last_award_date, primary_naics. MIT licensed. Updated monthly.

How often does the data update?

Incremental obligation pulls from USASpending Monday through Saturday. Idempotent re-ingestion Sundays. FedComp Index scores recalculate and the site regenerates after each pull. External mirrors (HuggingFace, Kaggle, PyPI) update monthly.

A cardinality gate rejects any pull where the contractor count drops more than 5% from the prior run, restoring from the last known-good snapshot. Prevents silent data loss from upstream API faults.

How does the proximity map apply to FAR compliance?

FAR Part 10 - Market Research. Contracting officers must identify capable sources before issuing solicitations. The proximity map provides a code-weighted, scale-filtered alternative to SAM.gov keyword search, surfacing firms by demonstrated obligation history under specific NAICS and PSC codes rather than self-reported registration claims.

FAR 19.702 - Subcontracting Plans. Prime contractors above the simplified acquisition threshold must set goals for 8(a), HUBZone, WOSB, SDVOSB, and VOSB subcontractors. Filtering the proximity map by certification type and posture class produces a provenance-verified candidate set that keyword search cannot approximate.

Posture Class 3 Teaming Strategy. The proximity map exposes which Posture Class 1 and Class 2 firms operate in the same NAICS corridor at 10×–100× scale. That magnitude asymmetry is the teaming aperture: those firms need small business subcontractors to satisfy FAR 19.702 goals, and the proximity map identifies the structural relationship.
